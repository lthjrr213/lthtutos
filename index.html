Algorithmique

Qu'est-ce qu'un algorithme ?
Suite finie et non-ambiguë d'instructions qui, à partir d'une entrée, produit une sortie en un temps fini. Un algorithme résout un problème de calcul bien défini.
ExempleRecette de cuisine : entrée = ingrédients, instructions = étapes précises, sortie = plat prêt. Ou : trouver le max dans [3, 7, 2, 9, 1] → parcourir, comparer → sortie : 9.
Pseudo-code
Langage de description d'algorithmes, indépendant de tout langage de programmation. Il utilise des structures proches du français et du code : si/alors/sinon, pour, tant que.
ExemplePOUR i de 1 à n FAIRE
  SI A[i] > max ALORS
    max ← A[i]
  FIN SI
FIN POUR
Invariant de boucle
Propriété vraie avant, pendant et après chaque itération. Utilisé pour prouver la correction d'un algorithme. Trois conditions : Initialisation, Conservation, Terminaison.
Exemple — Tri insertionInvariant : « A[1..i-1] est trié ».
Init : i=2, A[1..1] trié ✓
Conservation : on insère A[i] au bon endroit.
Terminaison : i=n+1 → A[1..n] trié ✓
Récursivité
Technique où une fonction s'appelle elle-même sur des sous-problèmes plus petits. Nécessite un cas de base et une réduction garantissant la terminaison.
Exemple — Factoriellefact(0) = 1 ← cas de base
fact(n) = n × fact(n-1) ← réduction
fact(4) → 4×3×2×1 = 24
Correction
Un algorithme est correct s'il s'arrête et produit le bon résultat pour toute entrée valide. On prouve la correction par invariants de boucle ou induction.
ExempleRecherche du max sur [3,7,2] :
• S'arrête après n=3 étapes ✓
• Retourne 7 (bien le max) ✓
→ algorithme correct pour tout tableau non vide.
Instance d'un problème
Une instance est une entrée particulière satisfaisant les contraintes du problème. Ex : trier [5,2,4,6,1,3] est une instance du problème du tri.
ExempleProblème : trouver le plus court chemin.
Instance : graphe G = {A→B:4, B→C:2, A→C:10}, départ A, arrivée C.
Solution : A→B→C = 6 (meilleur que A→C = 10).

Pseudo-code — Recherche linéaire & dichotomique
// Recherche linéaire — O(n)
RECHERCHE-LINEAIRE(A, v)
  pour i ← 1 à longueur[A]
    si A[i] = v
      retourner i
  retourner NIL

// Recherche dichotomique — O(lg n), tableau trié requis
RECHERCHE-DICHO(A, v, gauche, droite)
  si gauche > droite
    retourner NIL
  milieu ← ⌊(gauche + droite) / 2⌋
  si A[milieu] = v
    retourner milieu
  sinon si A[milieu] < v
    retourner RECHERCHE-DICHO(A, v, milieu+1, droite)
  sinon
    retourner RECHERCHE-DICHO(A, v, gauche, milieu-1)
Notation Θ (Theta)
Borne asymptotique exacte. f(n) = Θ(g(n)) signifie que f est bornée à la fois par en haut et en bas par g, aux facteurs constants près. Ex : tri par insertion = Θ(n²) au pire.
Exemplef(n) = 3n² + 5n + 2
→ Θ(n²) car 3n² ≤ f(n) ≤ 4n² pour n assez grand.
Le tri par insertion fait exactement Θ(n²) comparaisons au pire.
Notation O (Grand O)
Borne supérieure asymptotique. f(n) = O(g(n)) signifie que f ne croît pas plus vite que g. C'est la mesure de complexité dans le cas le plus défavorable.
ExempleRecherche linéaire dans [3,7,1,9,2] :
• Pire cas : élément absent → n comparaisons → O(n).
• On peut dire aussi O(n²) (borne valide, mais pas tight).
Notation Ω (Omega)
Borne inférieure asymptotique. f(n) = Ω(g(n)) signifie que f croît au moins aussi vite que g. Ex : tout algorithme de tri par comparaison est Ω(n lg n).
ExempleTri fusion sur n=8 éléments :
• Fait toujours ≥ n·lg(n)/2 comparaisons.
→ T(n) = Ω(n lg n) : on ne peut pas faire mieux en comparatif.
Cas favorable / défavorable
Le cas défavorable donne une garantie de performance. Le cas moyen est utile en pratique mais plus difficile à analyser. On privilégie souvent le pire cas.
Exemple — Tri rapideFavorable : pivot toujours médian → Θ(n lg n).
Défavorable : tableau déjà trié, pivot = min → Θ(n²).
Moyen (pivot aléatoire) : Θ(n lg n) en espérance.
Complexité spatiale
Quantité de mémoire utilisée en fonction de la taille n de l'entrée. Un algorithme en-place utilise O(1) mémoire auxiliaire (ex : tri par insertion).
ExempleTri fusion : copie L et R → O(n) mémoire auxiliaire.
Tri par insertion : échange en place → O(1) auxiliaire.
DFS récursif sur graphe : pile d'appels → O(V) espace.
Récurrences
Équations définissant T(n) en fonction de T sur des sous-problèmes. Méthodes de résolution : substitution, arbre récursif, théorème général (Master Theorem).
Exemple — Tri fusionT(1) = Θ(1)
T(n) = 2·T(n/2) + Θ(n)
→ Master Thm, cas 2 : T(n) = Θ(n lg n).

Pseudo-code — Théorème général (Master Theorem)

// Comparer f(n) avec n^(log_b a) :

// Cas 1 : f(n) = O(n^(log_b a - ε))  →  T(n) = Θ(n^(log_b a))
// Cas 2 : f(n) = Θ(n^(log_b a))     →  T(n) = Θ(n^(log_b a) · lg n)
// Cas 3 : f(n) = Ω(n^(log_b a + ε)) →  T(n) = Θ(f(n))

// Exemples :
// Tri fusion :  T(n) = 2T(n/2) + Θ(n)  → Cas 2 → Θ(n lg n)
// Recherche dicho : T(n) = T(n/2) + Θ(1) → Cas 2 → Θ(lg n)
// Strassen : T(n) = 7T(n/2) + Θ(n²)    → Cas 1 → Θ(n^2.81)
Tri par insertion — O(n²)
Construit le tableau trié élément par élément. Efficace pour les petits tableaux ou presque-triés. En-place. Stable. Analogue à trier des cartes à la main.
Exemple[5, 2, 4, 1]
i=2 : insérer 2 → [2,5,4,1]
i=3 : insérer 4 → [2,4,5,1]
i=4 : insérer 1 → [1,2,4,5] ✓
Tri par fusion — Θ(n lg n)
Diviser-pour-régner : divise le tableau en deux, trie récursivement, fusionne. Toujours Θ(n lg n). Stable. Nécessite O(n) mémoire auxiliaire.
Exemple[3,1,4,2]
Diviser : [3,1] | [4,2]
Trier : [1,3] | [2,4]
Fusionner : [1,2,3,4] ✓
Tri rapide — O(n lg n) moy.
Choisit un pivot, partitionne, trie récursivement. Cas moyen Θ(n lg n), cas défavorable O(n²). En-place. Non stable. Très efficace en pratique.
Exemple — pivot = dernier[3,6,8,10,1,2,1], pivot=1
Partition : [1,1 | 3,6,8,10,2]
Récursion sur chaque partie → tableau trié.
Tri par tas — Θ(n lg n)
Utilise une structure de tas (heap). Toujours Θ(n lg n). En-place. Non stable. Combine les avantages du tri par insertion (en-place) et du tri par fusion (n lg n).
Exemple[4,1,3,2,16,9,10]
1. Construire tas-max : [16,4,10,2,1,9,3]
2. Extraire max 16 → placer à la fin, répéter.
→ Résultat : [1,2,3,4,9,10,16] ✓
Tri par dénombrement — O(n+k)
Pour des entiers dans [0,k]. Compte les occurrences. Temps linéaire si k = O(n). Non comparatif. Stable. Nécessite O(k) mémoire.
Exemple — k=5A=[2,5,3,0,2,3,0,3]
Compter : C=[2,0,2,3,0,1]
Cumuler : C=[2,2,4,7,7,8]
→ B=[0,0,2,2,3,3,3,5] ✓
Borne inférieure Ω(n lg n)
Tout algorithme de tri par comparaison nécessite au moins Ω(n lg n) comparaisons dans le pire cas. Preuve par arbre de décision. Le tri par fusion est donc optimal.
Preuve intuitivePour trier n=3 éléments : 3! = 6 permutations possibles.
Arbre de décision a ≥ 6 feuilles → hauteur ≥ ⌈lg 6⌉ = 3.
→ Au moins 3 comparaisons nécessaires, confirme Ω(n lg n).
Pseudo-code — Tri par fusion
FUSION(A, p, q, r)
  n₁ ← q − p + 1 ;  n₂ ← r − q
  Créer tableaux L[1..n₁+1] et R[1..n₂+1]
  pour i ← 1 à n₁  faire  L[i] ← A[p+i−1]
  pour j ← 1 à n₂  faire  R[j] ← A[q+j]
  L[n₁+1] ← ∞ ;  R[n₂+1] ← ∞   // sentinelles
  i ← 1 ;  j ← 1
  pour k ← p à r
    si L[i] ≤ R[j]
      alors A[k] ← L[i] ;  i ← i+1
      sinon A[k] ← R[j] ;  j ← j+1

TRI-FUSION(A, p, r)
  si p < r
    alors q ← ⌊(p+r)/2⌋
          TRI-FUSION(A, p, q)
          TRI-FUSION(A, q+1, r)
          FUSION(A, p, q, r)
// T(n) = 2T(n/2) + Θ(n) → Θ(n lg n)
Pseudo-code — Tri rapide (Quicksort)
PARTITION(A, p, r)
  x ← A[r]          // pivot
  i ← p − 1
  pour j ← p à r−1
    si A[j] ≤ x
      alors i ← i+1
            échanger A[i] ↔ A[j]
  échanger A[i+1] ↔ A[r]
  retourner i+1

TRI-RAPIDE(A, p, r)
  si p < r
    alors q ← PARTITION(A, p, r)
          TRI-RAPIDE(A, p, q−1)
          TRI-RAPIDE(A, q+1, r)
// Cas moyen : Θ(n lg n) | Pire cas : O(n²) (tableau déjà trié)
Tableau (Array)
Accès direct en O(1) par indice. Insertion/suppression O(n). Taille fixe (statique) ou dynamique. Base de la plupart des structures de données.
ExempleA = [10, 20, 30, 40, 50]
A[2] → 30 en O(1) ✓
Insérer 15 en pos 1 : décaler A[1..4] → O(n).
Liste chaînée
Éléments reliés par pointeurs. Insertion/suppression en O(1) si pointeur connu. Accès en O(n). Simplement ou doublement chaînée. Pas d'accès aléatoire.
Exemple3 → 7 → 1 → 9 → NIL
Insérer 5 après 7 : changer pointeurs → O(1).
Trouver 9 : parcourir 3→7→1→9 → O(n).
Pile (Stack) — LIFO
Last In First Out. Opérations EMPILER et DÉPILER en O(1). Utilisée pour la récursion, l'évaluation d'expressions, l'annulation d'actions (Ctrl+Z).
Exemple — parenthèsesVérifier "(a+(b*c))" :
( → empiler ; ) → dépiler et apparier.
Pile vide à la fin → expression valide ✓
File (Queue) — FIFO
First In First Out. Opérations ENFILER et DÉFILER en O(1). Utilisée pour la gestion de tâches, BFS (parcours en largeur), tampons de communication.
Exemple — BFSGraphe : A–B, A–C, B–D
Enfiler A. Défiler A → enfiler B,C.
Défiler B → enfiler D. Défiler C, D.
Ordre visité : A, B, C, D ✓
Table de hachage
Accès, insertion, suppression en O(1) moyen. Utilise une fonction de hachage h(k) pour calculer l'indice. Gestion des collisions : chaînage ou adressage ouvert.
Exemple — m=7h(k) = k mod 7
Insérer 10 → h(10)=3, 22 → h(22)=1, 31 → h(31)=3.
Collision 10 et 31 en case 3 → chaînage : 3→[10→31].
Arbre binaire de recherche
Chaque nœud : sous-arbre gauche ≤ nœud < sous-arbre droit. Recherche, insertion, suppression en O(h). Si équilibré : h = O(lg n). Peut dégénérer en O(n).
ExempleInsérer 5,3,7,1,4 :
   5
 3  7
1 4
Rechercher 4 : 5→3→4 → trouvé en 3 étapes ✓
Tas (Heap)
Arbre binaire presque complet. Propriété tas-max : parent ≥ enfants. INSÉRER et EXTRAIRE-MAX en O(lg n). Base du tri par tas et des files de priorité.
Exemple — tas-maxTableau [16,14,10,8,7,9,3]
Parent 16 ≥ enfants 14,10 ✓
EXTRAIRE-MAX → 16, replacer 3 en racine, ENTASSER → [14,8,10,3,7,9] ✓
Graphe
Ensemble de sommets V et d'arêtes E. Représenté par liste d'adjacence (O(V+E)) ou matrice d'adjacence (O(V²)). Orienté ou non orienté, pondéré ou non.
ExempleV={A,B,C,D}, E={A–B, A–C, B–D, C–D}
Liste adj : A:[B,C], B:[A,D], C:[A,D], D:[B,C]
BFS depuis A → ordre : A,B,C,D.
Pseudo-code — Pile et File
// PILE (Stack) — LIFO
EMPILER(S, x)
  sommet[S] ← sommet[S] + 1
  S[sommet[S]] ← x

DÉPILER(S)
  si PILE-VIDE(S)  alors erreur "underflow"
  sommet[S] ← sommet[S] − 1
  retourner S[sommet[S] + 1]

// FILE (Queue) — FIFO
ENFILER(Q, x)
  Q[queue[Q]] ← x
  si queue[Q] = longueur[Q]
    alors queue[Q] ← 1
    sinon queue[Q] ← queue[Q] + 1

DÉFILER(Q)
  x ← Q[tête[Q]]
  si tête[Q] = longueur[Q]
    alors tête[Q] ← 1
    sinon tête[Q] ← tête[Q] + 1
  retourner x
Pseudo-code — Parcours de graphe (BFS & DFS)
// BFS — Parcours en largeur — O(V+E)
BFS(G, s)
  pour chaque sommet u ∈ V[G] \ {s}
    couleur[u] ← BLANC ; d[u] ← ∞ ; π[u] ← NIL
  couleur[s] ← GRIS ; d[s] ← 0 ; π[s] ← NIL
  Q ← ∅ ; ENFILER(Q, s)
  tant que Q ≠ ∅
    u ← DÉFILER(Q)
    pour chaque v ∈ Adj[u]
      si couleur[v] = BLANC
        couleur[v] ← GRIS ; d[v] ← d[u]+1 ; π[v] ← u
        ENFILER(Q, v)
    couleur[u] ← NOIR

// DFS — Parcours en profondeur — O(V+E)
DFS-VISITER(G, u)
  couleur[u] ← GRIS ; temps ← temps+1 ; d[u] ← temps
  pour chaque v ∈ Adj[u]
    si couleur[v] = BLANC
      π[v] ← u ; DFS-VISITER(G, v)
  couleur[u] ← NOIR ; temps ← temps+1 ; f[u] ← temps
Diviser pour régner
Diviser le problème en sous-problèmes similaires, résoudre récursivement, combiner. Ex : tri fusion, tri rapide, recherche dichotomique, algorithme de Strassen.
Exemple — Recherche dichotomiqueChercher 7 dans [1,3,5,7,9,11] :
Milieu=5 → 7>5 → chercher dans [7,9,11].
Milieu=9 → 7<9 → chercher dans [7].
Trouvé ! O(lg n) au lieu de O(n).
Programmation dynamique
Résout les problèmes à sous-structure optimale en mémorisant les solutions de sous-problèmes (mémoïsation ou tabulation). Ex : plus longue sous-séquence commune, sac à dos.
Exemple — Fibonaccifib(5) sans mémo : 15 appels récursifs.
Avec mémo : fib(2)=1, fib(3)=2, fib(4)=3, fib(5)=5.
→ 5 calculs seulement, O(n) au lieu de O(2ⁿ).
Algorithmes gloutons
À chaque étape, fait le choix localement optimal. Simple et efficace, mais ne fonctionne pas pour tous les problèmes. Ex : algorithme de Kruskal, de Dijkstra, rendu de monnaie.
Exemple — Rendu de monnaieRendre 41 cts avec {25,10,5,1} :
Glouton : 25¢ → reste 16¢ → 10¢ → reste 6¢ → 5¢ → 1¢.
4 pièces. Optimal ici ✓ (pas toujours le cas !).
Recherche exhaustive
Enumère toutes les solutions possibles. Garantit de trouver l'optimum mais souvent exponentiel. Utilisé quand n est petit ou comme base de comparaison.
Exemple — Sac à dos (n=3)Objets : {2kg/3€, 3kg/4€, 4kg/5€}, capacité=5kg.
Tester 2³=8 sous-ensembles → meilleur : {2kg,3kg}=7€.
Fonctionne pour n petit, impraticable pour n>30.
Problèmes NP-complets
Classe de problèmes pour lesquels aucun algorithme polynomial n'est connu. Ex : voyageur de commerce, problème du sac à dos, satisfaisabilité. On utilise des algorithmes d'approximation.
Exemple — TSP4 villes A,B,C,D : trouver le circuit le plus court.
Vérifier une solution : O(n) ✓ (NP).
Trouver l'optimale : (n-1)!/2 = 3 circuits pour n=4, mais 60 milliards pour n=14 !
Analyse amortie
Calcule le coût moyen d'une opération sur une séquence d'opérations. Méthodes : agrégat, comptable, potentiel. Utile pour tables dynamiques, tas de Fibonacci.
Exemple — Table dynamiqueDoubler la table quand pleine (coûts : 1,1,2,1,4,1,1,1,8,1…)
n insertions : coût total = O(n).
Coût amorti par insertion = O(n)/n = O(1) ✓
Pseudo-code — Programmation dynamique (Plus courte sous-séquence commune)
// LCS — Longest Common Subsequence — Θ(m·n)
LCS-LONGUEUR(X, Y)
  m ← longueur[X] ;  n ← longueur[Y]
  pour i ← 0 à m  faire  c[i,0] ← 0
  pour j ← 0 à n  faire  c[0,j] ← 0
  pour i ← 1 à m
    pour j ← 1 à n
      si xᵢ = yⱼ
        alors c[i,j] ← c[i−1,j−1] + 1 ;  b[i,j] ← "↖"
      sinon si c[i−1,j] ≥ c[i,j−1]
        alors c[i,j] ← c[i−1,j] ;  b[i,j] ← "↑"
      sinon  c[i,j] ← c[i,j−1] ;  b[i,j] ← "←"
  retourner c et b

// Dijkstra — O((V+E) lg V) avec tas de Fibonacci

Balises (Tags)
Les balises définissent les éléments. Elles s'ouvrent <tag> et se ferment </tag>. Certaines sont auto-fermantes comme <img /> ou <br />.
Exemple<p>Bonjour</p> — paragraphe ouvrant+fermant.

<h1>Titre</h1> → rendu : titre principal en gras.

Attributs
Les attributs fournissent des informations supplémentaires aux balises. Ex: <a href="url">, <img src="photo.jpg" alt="desc">
Exemple<a href="https://exemple.fr" target="_blank">Lien</a>

<input type="email" required placeholder="[email protected]">

DOM — Arbre
Le navigateur parse le HTML et construit un arbre d'objets appelé DOM (Document Object Model) que JS peut manipuler.
Exemple<body> → enfants : <h1>, <p>, <div>
JS : document.querySelector('h1').textContent = 'Salut';
→ modifie le titre en temps réel sans recharger la page.

Accessibilité

<img src="chat.jpg" alt="Chat roux endormi"> → description vocale.

<body>
  <div class="app">
    <h1>📝 Ma To-Do List</h1>
    <div class="input-row">
      <input type="text" id="taskInput" placeholder="Nouvelle tâche..."/>
      <button class="add" onclick="addTask()">Ajouter</button>
    </div>
    <ul id="taskList"></ul>
    <p class="count" id="count">0 tâche restante</p>
  </div>
  <script>
    const input = document.getElementById('taskInput');
    const list  = document.getElementById('taskList');
    const count = document.getElementById('count');
    let tasks = [];

    input.addEventListener('keydown', e => { if (e.key === 'Enter') addTask(); });

    function addTask() {
      const txt = input.value.trim();
      if (!txt) return;
      tasks.push({ text: txt, done: false });
      input.value = '';
      render();
    }
    function toggle(i) { tasks[i].done = !tasks[i].done; render(); }
    function remove(i) { tasks.splice(i, 1); render(); }
    function render() {
      list.innerHTML = tasks.map((t, i) => `
        <li class="${t.done ? 'done' : ''}">
          <input type="checkbox" ${t.done ? 'checked' : ''}
                 onchange="toggle(${i})">
          <span class="text">${t.text}</span>
          <button class="del" onclick="remove(${i})">✕</button>
        </li>`).join('');
      const rem = tasks.filter(t => !t.done).length;
      count.textContent = rem + (rem <= 1 ? ' tâche restante' : ' tâches restantes');
    }
  </script>
</body></html>
